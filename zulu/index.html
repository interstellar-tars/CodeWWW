How to Integrate Zulu SSO with Your Platform

Zulu SSO is a standards-compliant OAuth 2.0 and OpenID Connect (OIDC) provider. Follow these steps to integrate Zulu SSO with platforms like Google Workspace, Proton Mail, and others.

Step 1: Register Zulu as an Identity Provider

In the platformâ€™s admin console, locate the section for adding an external identity provider. Provide the following details:

  • Authorization Endpoint: https://your-zulu-sso-url/authorize
  • Token Endpoint: https://your-zulu-sso-url/token
  • Userinfo Endpoint: https://your-zulu-sso-url/userinfo
  • Scopes: openid profile email

Step 2: Configure Redirect URIs

In Zulu SSO, ensure that the redirect_uri matches the callback URL specified by the platform. For example:

https://your-platform.com/callback

Step 3: Test the Integration

Perform the following steps to test the integration:

  1. Initiate the login flow from the platform. It should redirect to the Zulu SSO login page.
  2. Log in using your Zulu SSO credentials.
  3. Verify that the platform receives a valid token and can fetch user information from the /userinfo endpoint.

Example Configuration for Google Workspace

In Google Workspace, navigate to Security > Set up single sign-on (SSO) and provide the following:

  • Sign-in page URL: https://your-zulu-sso-url/authorize
  • Sign-out page URL: https://your-zulu-sso-url/logout
  • Verification certificate: Upload your public key if required.
